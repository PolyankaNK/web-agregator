## Про проєкт

**Web Aggregator of Service Centers** — це дипломний вебпроєкт, мета якого полягає у створенні зручної платформи для пошуку сервісних центрів, перегляду інформації про них, ознайомлення з доступними послугами та цінами, а також взаємодії користувачів із сервісами через систему відгуків і список обраного.

Проєкт орієнтований на такі завдання:

- централізовано зібрати інформацію про сервісні центри;
- надати користувачу швидкий пошук і перегляд деталей;
- показувати галерею фотографій сервісного центру;
- відображати перелік послуг і діапазон цін;
- дозволити залишати оцінки та коментарі;
- дати змогу додавати сервісні центри в «Обране».

---

## Основний функціонал

### Для користувача

- головна сторінка з пошуком сервісних центрів;
- сторінка зі списком сервісів;
- сторінка деталей конкретного сервісного центру;
- перегляд опису, адреси, телефону, сайту та годин роботи;
- перегляд фотографій сервісного центру;
- перегляд доступних послуг;
- залишення відгуків з оцінкою за трьома критеріями:
  - обслуговування;
  - ціна;
  - якість;
- можливість додавати сервіси в «Обране»;
- перегляд власного списку обраних сервісів.

### Для адміністратора

- керування сервісними центрами через Django Admin;
- додавання категорій послуг;
- додавання послуг та цін;
- завантаження фото для сервісних центрів;
- модерація та наповнення каталогу.

### Для API

- JWT-автентифікація;
- отримання списку сервісних центрів у JSON;
- створення відгуків через API;
- додавання/видалення сервісів з обраного через API;
- перегляд обраних сервісів авторизованого користувача.

### Призначення основних модулів

#### `core`
Містить головну бізнес-логіку проєкту:
- моделі сервісних центрів;
- відгуки;
- фото;
- категорії;
- послуги;

- HTML-сторінки;

Містить логіку, пов’язану з користувачами:

Містить конфігурацію Django:
- `settings.py`
- `urls.py`
- `wsgi.py`
- `asgi.py`

## Моделі даних

У проєкті реалізовано такі основні моделі:

### `ServiceCenter`
Описує сервісний центр.

Поля:
- `name`
- `slug`
- `description`
- `city`
- `address`
- `phone`
- `website`
- `working_hours`
- `created_at`

### `ServiceCategory`
Категорія послуги.

Поля:
- `name`

### `ServiceCenterImage`
Фото сервісного центру.

Поля:
- `service_center`
- `image`
- `created_at`

### `ServiceOffer`
Послуга, яку надає сервісний центр.

Поля:
- `service_center`
- `category`
- `title`
- `description`
- `price_from`
- `price_to`
- `is_active`

### `Favorite`
Список обраного користувача.

Поля:
- `user`
- `service_center`
- `created_at`

### `ServiceReview`
Відгук користувача про сервісний центр.

Поля:
- `service_center`
- `user`
- `service_rating`
- `price_rating`
- `quality_rating`
- `comment`
- `created_at`

## Маршрути

### Вебмаршрути

- `/` — головна сторінка
- `/services/` — список сервісних центрів
- `/services/<slug:slug>/` — сторінка деталей сервісу
- `/favorites/` — сторінка обраних сервісів
- `/login/` — сторінка логіну
- `/register/` — сторінка реєстрації
- `/account/` — сторінка акаунта

### Системні маршрути

- `/admin/` — Django Admin
- `/api/token/` — отримання JWT access/refresh токенів
- `/api/token/refresh/` — оновлення access токена

---

## API endpoints

### Accounts API

- `POST /accounts/register/` — реєстрація користувача
- `POST /accounts/login/` — логін користувача
- `GET /accounts/me/` — дані поточного користувача

### Core API

- `GET /services/json/` — список сервісних центрів у JSON
- `POST /services/<slug:slug>/reviews/create/` — створення відгуку
- `POST /services/<slug:slug>/favorite/toggle/` — додати/видалити з обраного
- `GET /api/favorites/` — список обраних сервісів поточного користувача

## Локальний запуск

### 1. Клонувати репозиторій

```bash
git clone https://github.com/PolyankaNK/web-agregator.git
cd web-agregator
```

### 2. Створити та активувати віртуальне середовище

#### Windows

```bash
python -m venv venv
venv\Scripts\activate
```

#### Linux / macOS

```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Встановити залежності

```bash
pip install -r requirements.txt
```

### 4. Виконати міграції

```bash
python manage.py migrate
```

### 5. Створити суперкористувача

```bash
python manage.py createsuperuser
```

### 6. Запустити сервер

```bash
python manage.py runserver
```

Після запуску проєкт буде доступний за адресою:

```text
http://127.0.0.1:8000/
```

## Особливості реалізації

- `slug` для сервісного центру генерується автоматично;
- JWT використовується для API-автентифікації;
- зображення сервісних центрів зберігаються через Cloudinary;
- форма відгуків підтримує три окремі критерії оцінювання;
- користувач може формувати власний список обраних сервісів;
- проєкт має як HTML-інтерфейс, так і REST API.
